post

get /auth/login

  • Request
  • Response

Query Parameters

  • client_id: required (string)

    unique identifier provided by the UI client

  • ReturnTo: required (string)

    URL to return to after login or invite

  • ReturnToOnError: (string)

    URL to return to in case of expired invite

  • invite: (string)

    New user invite code

HTTP status code 302

On successful login or invite, a redirection to ReturnTo with state, token_type, expires_utc and access_token appended as query string parameters. If invite was provided, but it has expired, this response will be a redirection to ReturnToOnError with no query string parameters.

Headers

  • location: required (string)

    redirect URL

    Example:

    https://idp-pw.org/profile/intro?state=&token_type=Bearer&expires_utc=2018-12-20T18:41:52Z&access_token=hgEzk1kceC40FHeJm6cyfr5FgE9Tc4UH

HTTP status code 400

Error or missing information in request.

  • Request
  • Response

URI Parameters

  • resetId: required (string)

Body

Media type: application/json

Type: object

Properties
  • code: required (string)
  • client_id: required (string)

HTTP status code 200

Reset was validated. Response contains a 32-character access token. Subsequent API calls should use client_id concatenated with access_token in the Authorization header.

Body

Media type: application/json

Type: object

Properties
  • access_token: required (string)

HTTP status code 400

The provided code was incorrect or client_id was missing.

HTTP status code 410

The code has expired.
